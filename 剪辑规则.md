# 火柴人骨架剪辑规则

> 基于 MediaPipe PoseLandmarker 的人物骨架检测，自动剪辑视频的规则。

## 4条核心规则

### Rule 1 — 骨架扭曲检测（剪掉）

**不剪的正常动作：**
- 正常弯手（摸脸、挠头、挥手、招手）
- 转身/侧身（单侧关键点部分不可见）
- 抬腿/弯腿（走路、跑步、蹲下、跳跃）
- 手臂自然下垂（肩-肘-腕三点共线，角度接近0°）

**才剪的异常情况：**
- 关键点一帧内跳跃超过身体高度的40%以上，且躯干5点中3点以上同时跳跃 → 骨架数据飞了
- 双手同时举到肩膀以上且肘部弯到极限（角度<20°）→ 蜷缩姿态

实现方式：逐帧比较前后帧的关键点位移，躯干点（鼻、双肩、双髋）的帧间跳跃检测为主，极端的蜷缩姿态为辅。

### Rule 2 — 骨架不完整检测（剪掉）

**先判断全身/半身：**
- 扫描整个视频，统计脚踝（landmark 27,28）可见的帧占比
- 脚踝可见 > 30% → 全身（full），否则 → 半身（half）

**判断依据：**
- 全身：期望看到 鼻(0)+双肩(11,12)+双髋(23,24)+至少一脚踝
- 半身：期望看到 鼻(0)+双肩(11,12)+至少一手腕

**如果缺失期望的关键点 → 剪掉。**

### Rule 3 — 静止检测（剪掉）

- 鼻子(0)+双腕(15,16)的帧间位移 < 5像素
- 持续时间 > 1秒（30帧@30fps）
- → 判定为"静止"，剪掉

注意事项：正常动作中偶尔的卡顿（<1秒）不算静止。

### Rule 4 — 选最连贯片段

- 排除 Rule 1-3 标记的坏帧后，剩下的好帧中找**最长的连续好段**
- 从该段中间截取目标时长（自动避开头尾入画/出画）
- **时长分配规则：**
  - 给N个视频，需要合成长为T秒的成片
  - 每个视频保留时长 = T ÷ N
  - 例：3个视频剪成15秒 → 每个取5秒
  - 可选时长档位：3秒 / 5秒 / 8秒

## 参数速查

| 参数 | 默认值 | 说明 |
|:----|:------|:-----|
| fold_angle_threshold | 30° | 关节角度小于此值判定为折叠（当前主要用帧间跳跃检测，此项降级） |
| stillness_threshold | 5px | 帧间位移小于此值判定为静止 |
| stillness_max_seconds | 1.0s | 连续静止超过此秒数判定为坏段 |
| body_type_ankle_ratio | 0.3 | 脚踝可见帧占比超过此值判定为全身 |
| confidence_threshold | 0.5 | MediaPipe关键点置信度阈值 |
| jump_threshold | 0.4 | 关键点帧间位移超过身体高度此比例判定为数据异常 |

## 技术栈

- MediaPipe PoseLandmarker（33个关键点）
- OpenCV 逐帧处理 + VideoWriter 输出
- Python 3.11 纯 CPU 推理，实测 ~35 FPS
